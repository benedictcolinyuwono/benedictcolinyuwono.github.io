Project · ACS233 Burglar Alarm with Facial Recognition

Arduino (OOP C++), MATLAB (face recognition & UI), sensors/actuators integration

Scope & V-Model

  • Followed systems-engineering V-model: requirements → design (state/activity) → UML classes → OOP implementation → test.
  • Maintained traceability across documents and implementation (IDs referenced in commits/readme).

Core constraints implemented

  • PIN-controlled arming; status reporting to screen during execution.
  • Real hardware: PIR, magnetic switch, push button, buzzer, LEDs, solenoid lock; Arduino Mega 2560.
  • Webcam face recognition integrated via MATLAB with serial link to Arduino.

Evidence & Deliverables

  • Requirements & Design (≤16 pages) — stakeholder analysis incl. ED&I, user/system requirements (F/NF) with IDs, UML state & activity diagrams, and traceability table.
  • UML Class Diagram — classes/attributes/methods (+/−, types), multiplicities, relationships (incl. inheritance), cohesion/coupling noted; class–requirement traceability.
  • OOP Implementation — classes in Arduino sketch; MATLAB handles UI/PIN entry/face recognition; serial protocol ties states to actuators.
  • Testing — validates each requirement; notes for sensors’ debounce and PIR 8s high output behaviour.

Project · ACS6502/231 Robot Challenge

Mobile manipulator for tunnel navigation + line/ink task + marker drop

Challenge checkpoints covered

  • [1] start & move; [2] yaw rotate (optional); [3] remain in yellow zone to Right tunnel; [4] manipulation — draw line/leave ink on board; [5] pass Left tunnel; [6] halt; [7] drop marker; [8] Q&A readiness; [9] advanced mechanisms/control.

Compliance & design choices

  • Full autonomy during trials; untethered (battery powered); no PC/phone control.
  • Inverse kinematics used in arm control; no off-the-shelf chassis; parts built/assembled within budget.
  • Milestones tracked: design → electronics/control → first prototype → testing → demo.

Evidence & Deliverables

  • CAD overview and workspace/DOF notes; stability/CoM checks; fabrication log.
  • Electronics schematic & bill of materials (within per-member budget); battery selection rationale.
  • Control stack (navigation + manipulation), parameters, and IK notes; test videos.
